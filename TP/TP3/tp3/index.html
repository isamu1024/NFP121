Tout ici reprend ce qui a été vu précédemment mais arrêtons nous sur la méthode public boolean equals(Object o)

Afin de mettre en application ce qui a été vu en ED, je choisis ici de créer un iterator
pour parcourir la pile. C'est une méthode un peu surdimensionnée ici car on pourrait parfaitement accéder aux index dans une boucle for.

On vérifie dans un premier temps si l'objet passé en paramètre est de l'instance Pile et si oui, on cast.

On fait des vérifications d'usage sur la taille, la capacité et si les 2 piles sont vides

Si elles ne sont pas vides, les vérifications précédentes nous assure qu'elle sont de meme taille, donc on peut iérer et comparer chaque index sans risque d'appeler un index non affecté ou inexistant.

Appeler la méthode hashCode() de la classe Object nous permet d'attribuer un hash spécifique à chaque object mais identique quelque soit l'instance (voir doc).
Ma première idée fut d'appeler le hashCode fourni, mais celui-ci est spécifique à la Pile, pas à l'object contenu a un index donné !
de plus elle appelle dans le corps de la méthode toString() ! Un entier va donc renvoyer le hash de son équivalent converti en String !
générant de faux positifs entre des valeurs de type Integer et String, pas vraiment ce que l'on cherche à avoir !

Question 2.5

Le couple equals, hashCode

Que pensez vous de cette réponse pour les classes Pile, Pile2, Pile3 et Pile4 ?

public boolean equals(Object o) {
        if (o instanceof PileI) {
          PileI p = (PileI)o;
          return this.capacite() == p.capacite()
              && this.hashCode() == p.hashCode();
        } else
          return false;
      }

Cette solution est incorrecte pour 3 raisons :

  • En premier lieu nous testons si l'Objet passé en paramètre est une instance de l'interface PileI
    C'est le cas de nos 4 piles, donc une Pile4 passée en paramètre d'une instance Pile sera
    considérée comme valide ! Hors 2 object peuvent implanter une même interface sans pour autant être semblable.
  • Cette méthode ne compare pas la taille de la pile, uniquement sa capacité, une pile avec 2 éléments sera traitée comme
    équivalente à une pile avec 3 éléments.
  • Enfin, en utilisant la méthode hashCode() de la pile et non celle des éléments contenus dans la pile. Une pile contant des String
    renverra une égalité si on la compare avec une pile contenant des int, car l'implantation de la méthode hashCode proposée procède en
    premier lieu à une conversion en String de la pile. Une pile p1 = [1,2,3] sera considérée comme égale à une pile p2 = ["1","2","3"]

Cette autre réponse, est élégante, mais n'est pas correcte

public boolean equals(Object o) {
        return this.toString().equals(o.toString());
      }

Cette méthode compare uniquement la valeur retournée par la méthode toString() de chaque pile, elle ne garantie en rien que le type des objets contenus soient équivalent.
on pourrait parfaitement redéfinir les méthodes toString() des objets contenus pour leur faire retourner ce que l'on souhaite et générer de faux positifs.

        public class Pile2<T> implements PileI<T> {
            /** par délégation : utilisation de la class Stack */
            private Stack<T> stk;
            /** la capacitÃ© de la pile */
            private int capacite;
        
            /**
             * Création d'une pile.
             * 
             * @param taille la "taille maximale" de la pile, doit Ã¨tre > 0
             */
            public Pile2(int taille) {
                if (taille <= 0)
                    this.capacite = CAPACITE_PAR_DEFAUT;
                else
                    this.capacite = taille;
        
                stk = new Stack<T>();
        
            }
        
            public Pile2() {
                this(0);
            }
        
            @Override
            public void empiler(T o) throws PilePleineException {
                if (stk.size() == this.capacite)
                    throw new PilePleineException();
                this.stk.push(o);
            }
        
            @Override
            public T depiler() throws PileVideException {
                if (stk.size() == 0)
                    throw new PileVideException();
                return stk.pop();
            }
        
            @Override
            public T sommet() throws PileVideException {
                if (stk.size() == 0)
                    throw new PileVideException();
                return stk.peek();
            }
        
            @Override
            public int capacite() {
                return this.capacite;
            }
        
            @Override
            public int taille() {
                return stk.size();
            }
        
            @Override
            public boolean estVide() {
                return stk.empty();
            }
        
            @Override
            public boolean estPleine() {
                return stk.size() == this.capacite;
            }
        
            public String toString() {
                String s = "[";
                for (int i = taille() - 1; i >= 0; i--) {
                    s += stk.get(i).toString();
                    if (i != 0)
                        s += ", ";
                }
